# Challenge 4  - Building Recon

**Points:** 350  
**Protocol:** BACnet/IP  
**Difficulty:** Intermediate  

---

## The Story

Remember that device at 10.0.1.20 from your very first scan? It had no TCP ports open  - but it's definitely there. Something is monitoring the building's environment. Not all industrial protocols use TCP.

## What You Know

- 10.0.1.20 is on the corporate network (same as you)
- No TCP ports were found in earlier scans
- OPC-UA (CH-02) showed "BuildingManagement" sensor data  - temperature, humidity

## Step 1: Think Beyond TCP

Your earlier scans used `-sT` which only scans TCP. Many industrial protocols use **UDP** instead.

**Hint:** Building automation commonly uses UDP port 47808.

<details>
<summary>How to scan UDP?</summary>

```bash
sudo nmap -sU -p 47808 10.0.1.20
```
Note: UDP scans require `sudo`.
</details>

## Step 2: What is BACnet?

BACnet (Building Automation and Control Networks) controls:
- HVAC (heating, ventilation, air conditioning)
- Lighting
- Access control (door locks)
- Fire alarm systems

It uses UDP port 47808 and has **zero authentication** by default.

## Step 3: Discover the Device

Use the BACnet tool to find out what's running.

<details>
<summary>How?</summary>

```bash
python3 ~/tools/bacnet-tool.py -t 10.0.1.20 -c scan
```
</details>

## Step 4: Enumerate Objects

BACnet organizes data as objects:
- **AI** (Analog Input)  - sensors like temperature, humidity
- **AV** (Analog Value)  - configuration parameters
- **BI** (Binary Input)  - on/off states

List everything on the device.

<details>
<summary>Command?</summary>

```bash
python3 ~/tools/bacnet-tool.py -t 10.0.1.20 -c list
```
</details>

**Look carefully at the output.** Some objects have unusual instance numbers  - those are worth investigating.

## Step 5: Dig Deeper

A thorough pentester doesn't just read values  - they check all properties of interesting objects. BACnet objects have properties like Present-Value (85), Description (28), and Object-Name (77).

**Hint:** An object with an unusual instance number might have something hidden in its Description.

<details>
<summary>How to read all properties?</summary>

```bash
python3 ~/tools/bacnet-tool.py -t 10.0.1.20 -c read-all --type AV --instance <number>
```
Try different instances, especially unusual ones.
</details>

---

## What You Learned

- BACnet uses UDP  - TCP scans miss it completely
- Building management systems often sit on the corporate network, not segmented
- BACnet has zero authentication  - anyone can read AND write
- An attacker could control HVAC, unlock doors, disable fire alarms
- Always scan UDP ports when testing OT environments
