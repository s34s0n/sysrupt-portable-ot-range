# Challenge 1  - Perimeter Breach

**Points:** 100  
**Protocol:** HTTP  
**Difficulty:** Beginner  

---

## The Story

You just plugged your laptop into a corporate network at a water treatment facility. You have no credentials, no map, no idea what's inside. Your job: find a way in.

## What You Know

- You have an Ethernet connection
- Your IP is in the 10.0.1.x range
- Something is running on this network

## Step 1: Reconnaissance

Every pentest starts with scanning. What devices are on this network? What services are running?

**Hint:** Use `nmap` to scan the corporate subnet.

<details>
<summary>Need help with the command?</summary>

```bash
nmap -sT 10.0.1.0/24
```
</details>

**What to look for:** Open ports, especially web servers (port 80, 8080).

## Step 2: Explore the Web Application

You found a web server. Open it in your browser.

**What to look for:**
- Is there a login page?
- Check the page source (Ctrl+U)  - developers sometimes leave comments
- Are there any API endpoints?

## Step 3: Find a Vulnerability

Web applications often have flaws in how they handle user data. Try accessing resources you shouldn't be able to.

**Hint:** What happens if you change numbers in the URL?

<details>
<summary>Stuck?</summary>

Try accessing employee records by ID:
```
http://<ip>/api/employee/1
http://<ip>/api/employee/2
```
This is called IDOR (Insecure Direct Object Reference).
</details>

## Step 4: Get Access

You found some data. Can you use it to log in?

**Hint:** Look for usernames and try common passwords.

<details>
<summary>Still stuck?</summary>

Try common credentials: admin/admin, admin/admin123, admin/password
</details>

## Step 5: What Did You Find?

Once you're in, explore everything. The admin panel contains critical information about the internal network that you'll need for every challenge that follows.

**Write down all the IPs and services you see.**

---

## What You Learned

- Network scanning reveals hidden services
- Web applications often have IDOR vulnerabilities
- Default/weak credentials are everywhere
- One compromised system reveals the entire network map
